# URANGO 프로젝트 종합 지식베이스

## 📋 프로젝트 개요

### 기본 정보
- **프로젝트명**: URANGO (유랑고)
- **의미**: "우리랑 Go" - 함께 가는 아웃도어 활동
- **목적**: 아웃도어 액티비티 소셜 플랫폼 개발
- **플랫폼**: iOS, Android 모바일 애플리케이션
- **개발 기간**: 8.5개월 (2026년 3월 ~ 11월)
- **예산**: 431,300,000원 (VAT 별도)

### 핵심 가치
1. **안전성**: 검증된 리더와 평가 시스템
2. **접근성**: 초보자도 쉽게 참여 가능
3. **확장성**: 4개 카테고리에서 시작, 향후 확장
4. **수익성**: 광고 및 제휴 기반 비즈니스 모델

---

## 🚴 지원 카테고리

### 1. Cycle (사이클)
- 자전거 라이딩
- 제휴: 자전거 용품점, 로라방, 버스 운송 회사

### 2. Bike (바이크)
- 오토바이 투어링
- 제휴: 오토바이 용품점

### 3. Run (러닝)
- 마라톤 & 러닝
- 제휴: 마라톤 용품점

### 4. Mountain (등산)
- 등산 & 트레킹
- 제휴: 버스 운송 회사, 산악용품점
- 특별 기능: 산악협회 연동

## 🎯 주요 기능 명세

### 사용자 인증 및 관리
- 이메일 기반 회원가입/로그인
- 프로필 사진 업로드 및 관리
- 개인 정보 및 활동 이력 관리

### 리더(Leader) 기능

#### 활동 생성
**When (일정)**
- 연/월/일 선택

**Where (장소)**
- 국가 선택
- 지역(Province) 선택
- 상세 코스 입력

**Attendants (참가자 설정)**
- 난이도 선택:
  - Master (고급)
  - Intermediate (중급)
  - Beginner (초급)
- 인원 제한:
  - 2-5명
  - 6-10명
  - 11명 이상

#### 관리 기능
- 참가 신청자 승인/거부
- 활동 중 실시간 위치 공유
- 참가자 관리

### 팔로워(Follower) 기능

#### 검색 및 필터
**날짜 필터**
- 6개월
- 3개월
- 1개월
- 3주
- 2주
- 1주
- 즉시참여(Pop join)

**장소 필터**
- 국가/지역/상세지역 선택

**인원 필터**
- 2-5명 / 6-10명 / 11명 이상

#### 참여 기능
- 활동 검색 및 상세 보기
- 참가 신청
- 리더 팔로우

### 소셜 기능
1. **Follow/Follower 시스템**: 사용자 간 연결
2. **Direct Message (DM)**: 1:1 실시간 채팅
3. **사진첩**: 개인별/단체별 앨범 관리 (IDC 연동)
4. **Contact 관리**: 연락처 통합 관리
5. **게시판**: 커뮤니티 소통 (2단계: 회원 간 거래 기능 추가)

### 평가 및 랭킹 시스템
- 리더 피드백 (별점 등)
- 랭커 생성 및 관리
- 신뢰도 기반 매칭

### 위치기반서비스 (LBS)
- 실시간 GPS 위치 추적
- 활동 경로 기록 및 공유
- 지도 기반 활동 검색

### 광고 및 제휴
- 카테고리별 제휴 업체 광고 노출
- 광고 컨셉 판매 기능

## 🛠️ 기술 스택

### Frontend
- **Framework**: React Native
- **이유**: iOS/Android 동시 개발, 비용 절감, 빠른 개발 속도

### Backend
- **Runtime**: Node.js
- **Framework**: Express
- **이유**: 빠른 개발 속도, 실시간 처리 우수

### Database
- **Main DB**: PostgreSQL (관계형 데이터 처리)
- **Cache**: Redis (성능 향상)

### Cloud Infrastructure
- **Provider**: AWS
- **Services**: 
  - EC2 (서버)
  - RDS (데이터베이스)
  - S3 (이미지/파일 저장)

### External Services
- **Maps**: Google Maps API
- **Real-time**: Socket.io
- **Push**: Firebase Cloud Messaging
- **Auth**: OAuth 2.0 + JWT

### System Architecture
- 3계층 아키텍처
  - Presentation Layer: React Native 앱
  - Business Logic Layer: Node.js REST API
  - Data Layer: PostgreSQL + Redis
